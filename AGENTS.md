# AGENTS

## Project Overview
This repository is an AI data operations platform workspace.

## Working Principles
- Keep changes small, focused, and well-documented.
- Prefer clear directory structure and maintainable code.
- Use existing conventions when present.
- Verify important changes with the relevant command or test.

## Repository Structure
- docs/ - product requirements, architecture, database, and API docs
- backend/ - backend services and APIs
- frontend/ - frontend application code
- rules/ - project rules and policies
- prompts/ - prompt templates and examples
- workflows/ - workflow definitions
- tests/ - test suites
- scripts/ - utility scripts

## Expectations for Agent Changes
- Do not remove or overwrite unrelated files without reason.
- Preserve existing functionality unless explicitly asked to change it.
- When adding new functionality, include reasonable documentation or comments.
- If a task is ambiguous, ask for clarification rather than making risky assumptions.
